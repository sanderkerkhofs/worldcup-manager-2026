01 - Architecture

1. High-Level Architecture

  • Frontend (Next.js) renders pages and calls API endpoints.
  • Backend (Express) exposes role-based endpoints and business rules.
  • Database (PostgreSQL + Prisma) stores persistent competition data.

2. Layered Backend Design

  • Controller layer: HTTP handling, auth/role checks, service delegation.
  • Service layer: business workflows and knockout validation rules.
  • Model layer: domain validation and safe mapping.
  • Repository/Prisma layer: database access and persistence.

This separation keeps logic testable and maintainable.

3. Frontend Design

  • pages/: route-level pages
  • components/: reusable UI
  • services/: API call wrappers
  • lib/: session and utility helpers
  • styles/: global CSS

4. Cross-Cutting Concerns

Authentication and Authorization

  • JWT issued on login.
  • Protected calls send token.
  • Backend middleware enforces roles.

Error Handling

  • Services throw typed errors.
  • Global error middleware formats responses consistently.

API Documentation

Swagger is served at /api-docs and acts as live API contract.

5. Request Lifecycle Example

For PATCH /api/matches/:matchId/status:

  1. Request enters match controller.
  2. JWT is validated.
  3. Role check allows ADMIN or REFEREE.
  4. Service validates business rules.
  5. Prisma updates database.
  6. Controller returns JSON response.

6. Why It Fits This Project

  • Strict knockout rules are centralized in services.
  • Role-based permissions are explicit and secure.
  • Typed frontend services simplify UI integration.
  • Prisma + seed script support repeatable demos/tests.
