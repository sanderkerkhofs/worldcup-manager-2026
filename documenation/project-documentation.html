Project Documentation

Tournament Manager

Tournament Manager is a full-stack learning project for managing a football knockout tournament, with role-based workflows for admin, coach, and referee users.

1. Technology Stack

Backend

  • Node.js + Express + TypeScript
  • Prisma ORM + PostgreSQL
  • JWT authentication and role-based authorization
  • Swagger/OpenAPI documentation

Frontend

  • Next.js (Pages Router) + React + TypeScript
  • SWR for API data fetching

2. Project Structure

  • analysis/: planning and school deliverables
  • back-end/: API, services, models, Prisma, seed scripts
  • front-end/: pages, components, client services, styles
  • docker-compose.yml: local PostgreSQL setup

3. Core Domain

Entity Description
UserSystem account with role ADMIN, COACH, REFEREE, or GUEST.
TeamNational team with country metadata and assigned coach.
PlayerTeam player with availability state.
MatchFixture between teams with score, referee, status, and embedded stage metadata.
GoalScoring event linked to match, player, and minute.

Match statuses: NOT_STARTED, IN_PROGRESS, COMPLETED

4. Role-Based Access

  • ADMIN: manages users, initiates/simulates stages, edits matches.
  • COACH: updates own team player availability.
  • REFEREE: updates assigned matches and goals.
  • GUEST: public access to non-protected views.

5. Main Workflows

Competition progression

  1. First-stage matches are seeded as IN_PROGRESS.
  2. Referees/Admin update scores and goals.
  3. Matches are completed.
  4. Next-stage matches are auto-filled with winners.
  5. Next-stage matches become IN_PROGRESS automatically.

Match editing validation

  • Match teams must be known for active editing.
  • Knockout matches cannot end in a draw.
  • Only available players can be selected as scorers.

6. API Summary

Base URL: http://localhost:3000

Area Key Endpoints
Auth POST /api/auth/register, POST /api/auth/login, GET /api/auth/me
Competition GET /api/competition/overview, GET /api/competition/rounds, POST /api/competition/rounds/:roundId/simulate
Teams GET /api/teams, GET /api/teams/:teamId, GET /api/teams/:teamId/players, admin CRUD
Players GET /api/players, GET /api/players/:playerId, admin CRUD, PATCH /api/players/:playerId/status
Matches GET /api/matches, GET /api/matches/:matchId, status/result/goals updates
Users GET /api/users, DELETE /api/users/:userId (admin only)

Swagger UI: http://localhost:3000/api-docs

roundId in competition routes is a stage identifier, not a persisted Round entity id.

7. Local Setup

Start database

docker compose up -d postgres

Backend

cd back-end
npm install
npm run db:generate
npm run db:seed
npm start

Frontend

cd front-end
npm install
npm run dev
