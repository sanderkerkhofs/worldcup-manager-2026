05 - Workflows and Sequence Diagrams

1. Login Flow

User -> Frontend -> POST /api/auth/login -> Backend -> DB lookup (case-insensitive)
Backend -> token + user payload -> Frontend stores session

2. Initiate Round (Admin)

Admin clicks initiate
Frontend calls POST /api/competition/rounds/:id/initiate
Backend validates previous round + team assignment
Backend updates matches: NOT_STARTED -> IN_PROGRESS
Backend returns updated round/matches

3. Referee Match Update

Referee edits status/score
Frontend calls PATCH/PUT match endpoints
Backend validates role + assignment + knockout rules
DB updates match/goals
Frontend receives updated match

4. Coach Availability

Coach toggles player status
Frontend PATCH /api/players/:id/status
Backend validates team scope
DB updates availability
Frontend updates player state

5. Bracket Progression

  1. Round exists
  2. Admin initiates round
  3. Matches become IN_PROGRESS
  4. Results and goals are entered
  5. Matches become COMPLETED
  6. System validates readiness for next stage
